Información del Servidor


Resumen del proyecto

Con el avance de la tecnología en los últimos años nos ha permitido el desarrollo de varios dispositivos que facilitan las actividades diarias, dando acceso a la información de manera casi inmediata. Un aspecto importante al momento de trasmitir un mensaje es garantizar que sean seguros y que no pueda ser manipulados por personas que nos estén autorizadas. A diario se presentan cientos de ataques de intrusión que busca dañar esta información mediante diferentes tipos de ataques que en su gran mayoría busca dañar un servicio o retener la información a cambio de dinero. Esta necesidad ha incentivado el desarrollo de tecnologías para la protección y detección de amenazas, entre ellas se encuentran los algoritmos de aprendizaje autónomo que son capaces de detectar patrones para determinar si son posibles amenazas. Se propone el desarrollo de este modelo mediante el uso de Deep learning con el objetivo de poder detectar los posibles ataques y así disminuir la propagación de los mismo en dispositivos de IoT

Objetivos

Objetivo General


Implementar un algoritmo evolutivo para optimizar los pesos de una red neuronal de aprendizaje profundo para identificar y clasificar posibles ataques de intrusión utilizando el conjunto de datos “Detección de ataques de intrusión IoT” del repositorio KRAGGLE.

Objetivos Específicos


1. Preparar y procesar el conjunto de datos destinado al entrenamiento de la red neuronal, seleccionando las características relevantes mediante un método de reducción de dimensionalidad.

2. Desarrollar una interfaz de usuario que permita la operación de la red neuronal, a partir de la configuración de parámetros específicos para la simulación de ataques.

3. Medir el desempeño del modelo con métricas específicas como exactitud, precisión, sensibilidad.

Tipos de ataques

1. Ataques de Denegación de Servicio Distribuida (DDoS):

Estos ataquesse ejecutan desde múltiples dispositivos comprometidos, creando un volumen de tráfico tanalto que el objetivo se vuelve incapaz de responder a solicitudes legítimas.

• Inundación RSTFIN (DDoS): Este ataque utiliza paquetes TCP con las banderas RST (Restablecer) y FIN (Finalizar). El RST se usa para cerrar conexiones y el FIN para finalizar la transmisión de datos. Al enviar paquetes con estas banderas a alta velocidad, el servidor objetivo se ve abrumado al tener que manejar y procesar estos paquetes, lo que puede resultar en una interrupción de servicio.

• Inundación PSHACK (DDoS): En este ataque, se envían paquetes TCP con las banderas PSH (Push) y ACK (Acknowledgement). La bandera PSH solicita que los datos se envíen inmediatamente sin esperar a que el búfer se llene, mientras que ACK confirma la recepción. Esta combinación genera un alto volumen de tráfico urgente que consume los recursos del servidor.

2. Ataques de Denegación de Servicio (DoS):

Similar a los ataques DDoS, pero originadosdesde un solo dispositivo, lo que limita su escala.
• Inundación UDP (DoS): Bombardea al objetivo con paquetes UDP desde una única IP. Aunque similar a la inundación UDP de DDoS, el ataque proviene de un solo dispositivo.
• Inundación SYN (DoS): Similar a la inundación SYN en DDoS, pero originada desde una única IP. Ataca el proceso de establecimiento de conexión TCP.
• Inundación TCP (DoS): Similar a la inundación TCP en DDoS, pero realizada desde un único host, saturando la capacidad de conexión del servidor.
• Inundación HTTP (DoS): Envía una alta cantidad de solicitudes HTTP a un servidor, con el objetivo de agotar sus recursos y hacer que no pueda atender solicitudes legítimas. Esto puede incluir solicitudes GET y POST.


3. 3. Ataques de Reconocimiento:

Etapa inicial en la recopilación de información sobre un objetivo.
• Barrido de Ping: Envía paquetes ICMP de solicitud de eco a un rango de direcciones IP para identificar dispositivos activos en una red.
• Escaneo del Sistema Operativo: Utiliza sondas y técnicas para deducir el sistema operativo de los dispositivos objetivo, lo que puede ayudar a identificar vulnerabilidades específicas del sistema.
• Escaneo de Puertos: Escanea una red o sistema para identificar puertos abiertos y servicios disponibles, buscando posibles vulnerabilidades en los servicios expuestos.
• Escaneo de Vulnerabilidad: Emplea herramientas automatizadas para identificar fallos de seguridad, como software desactualizado, configuraciones incorrectas o contraseñas débiles.
